# steno-ios

iOS app for [Steno](https://github.com/parisyee/steno-cloud). Receives voice notes via the iOS share sheet (from Voice Memos or any app that exports audio), sends them to the Steno API for transcription, and displays the results. Direct in-app recording is planned but not yet implemented.

## Architecture

```
Voice Memos (or any source of audio)
      │
      │  iOS Share Sheet
      ▼
StenoShareExtension         ← activates on public.audio UTIs
      │
      │  copies .m4a to App Group container, opens steno://transcribe
      ▼
Steno (main app)            ← reads file, POSTs to API, shows result
      │
      │  POST /transcribe (multipart, Bearer auth, 600s timeout)
      ▼
Steno API (Cloud Run)
https://steno-836899141951.us-central1.run.app
```

## First-time setup

```bash
brew install xcodegen
cp Shared/Secrets.swift.example Shared/Secrets.swift
# edit Shared/Secrets.swift — paste the real STENO_API_KEY
xcodegen generate
open Steno.xcodeproj
```

In Xcode:

1. Select the **Steno** target → **Signing & Capabilities** → pick your team.
2. Repeat for the **StenoShareExtension** target (same team).
3. Plug in your iPhone, select it as the run destination, hit **Cmd+R**.
4. On first install: Settings → General → VPN & Device Management → trust your profile.

With a free Apple Developer account the signature expires every 7 days — just re-run from Xcode to refresh. With a paid account ($99/yr) it persists.

## Daily development

- Change anything in `project.yml`, or add/remove files: `xcodegen generate`, then reopen Xcode.
- Change Swift source only: Xcode picks it up — no regeneration needed.
- Build from the command line: `xcodebuild -scheme Steno -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO build`.

## Configuration

Non-secret values live in [`Shared/Config.swift`](Shared/Config.swift): API base URL, endpoint path, App Group ID, shared filename.

The API key lives in [`Shared/Secrets.swift`](Shared/Secrets.swift.example) (gitignored) as an extension on `Config`. If the file is missing the build fails with "no member named `apiKey`" — that's the signal to copy the `.example` template.

To change the bundle ID prefix (currently `com.parisyee`), edit `options.bundleIdPrefix` in `project.yml`, update `appGroupID` in `Config.swift` + both `.entitlements`, then regenerate.

## How the share flow works

1. User taps share on a voice memo in Voice Memos.
2. `StenoShareExtension` appears because the `.m4a` conforms to `public.audio`.
3. Extension copies the file to `Config.sharedFileURL` (App Group container).
4. Extension opens `steno://transcribe` — iOS brings the main app forward.
5. Main app's `onOpenURL` fires → `TranscriptionStore.processSharedFile()` reads the file, deletes it from the shared container, and POSTs to `/transcribe`.
6. Response `{text, id}` is displayed and persisted in `UserDefaults(suiteName:)` (App Group).

The upload path is also exposed as `TranscriptionStore.transcribe(audioFileURL:)` for future in-app recording.

## API contract

**Request:** `POST /transcribe`

```
Authorization: Bearer <STENO_API_KEY>
Content-Type: multipart/form-data
file: <audio.m4a>
```

**Response:**

```json
{ "text": "...", "id": "uuid" }
```

Timeout is 600s — long recordings can take several minutes.
